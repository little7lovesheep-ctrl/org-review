填写规范
  • 每个问题尽量按“结论 + 数据/事实 + 下一步”填写。
  • 不要写“持续推进、加强协同、优化流程”这类空话。

目标完成与核心结果

参考示例写一段核心结论即可。尽量带数字，但不用拆成很多字段。

业务线填写示例
  • 会员收入：目标 240 万，实际 218 万，完成率 90.8%，判断为基本达成。
  • 营销回款：目标 150 万，实际 120 万，完成率 80%，需说明差距来自账期还是项目未执行。
  • 港口货源付费：目标 100 单，实际 25 单，完成率 25%，判断为未达成但供给侧已验证。
平台支撑岗位填写示例
  • 技术：上半年支撑会员、货源、救援、营销上线 X 个需求，其中 X 个直接服务收入或客户交付；主要差异来自需求变更/排期冲突/技术债。
  • 数据：上半年完成 X 次数据导出/分析/监控，支撑会员分层、港口货源漏斗、救援结算等决策；差异来自数据口径不统一或字段缺失。
  • 内容：上半年发布 C 端司机内容 X 条，有效播放 X，沉淀可用于客户/会员/救援的素材 X 条；B 端客户案例内容不足，下半年需要补客户证据类内容。
  • 客服/私域：上半年处理客诉/回访/企微触达 X 次，沉淀用户反馈 X 条，支撑会员续费、货源反馈或救援服务稳定。
支撑岗位不强行填收入，但要说明支撑了哪条业务线，以及产出是否改善了效率、交付、数据决策、用户反馈或客户证据。

  • 问题：写影响业务结果的阻碍，不写个人情绪。
  • 每类建议最多写2条。

创新尝试与协同支持

只收已经尝试过的事。没有结果的想法不要写成创新，写到下半年目标里即可。

判断标准
  • 有效创新：节省时间、带来收入、改善转化、降低人工、形成可复用方法。
  • 有效协同：明确谁支持了什么，解决了什么阻碍。

关键协同，最多2条

下半年目标规划

下半年目标必须从上半年结果里推出来：哪些继续做、哪些要修正、哪些要停止、哪些要新增。

先看四条业务线核心经营指标
  • 会员：收入目标、续费率、新开会员数、退款率、私域触达转化率。
  • 营销/商业：合作金额、执行金额、回款金额、毛利、重点客户续约/复购数量、B端客户案例内容、客户素材/案例沉淀、线索或私域导流。
  • 救援：结算额、毛利额、毛利率、订单量、线上新增结算门店、新客户转化、核心客户续约、新城市/门店覆盖。
  • 货源：付费收入、付费订单数、拨打UV、拨打转化率、新增会员数、用户反馈采集量。
  • 企微/私域运营：可触达用户数、企微关联率、标签准确率、触达批次数、回复率、转化/激活人数、用户反馈采集量、小宝号管理覆盖率。
  • 内容/短视频：当前更偏C端司机内容，需单独记录发布量、有效播放、互动、私域导流；B端客户内容偏少，若服务营销线，重点看客户案例、项目证据、品牌素材沉淀。
承接关系写法
  • 继续：上半年有效的动作，下半年扩大或固化。
  • 修正：上半年有结果但不稳定，下半年改口径、流程、资源或目标。
  • 停止：上半年验证无效或投入产出差，下半年不再消耗资源。
  • 新增：上半年暴露的新机会或新风险，下半年设置新目标。
